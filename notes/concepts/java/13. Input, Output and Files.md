
Input/Output makes a program more dynamic. Users can interact directly with the program, and the output changes depending on the input. By saving data to a file, the data can persist after the program terminates.

## Scanner

The `Scanner` class is a useful tool in Java that can read data from various input sources, such as the keyboard or files. `Scanner` reads input data sequentially and converts it into the required data type, such as an integer, floating-point number, or string.

A `Scanner` can be used simply by creating a `Scanner` object and passing `System.in` as an argument. `System.in` is the standard input stream. When it is passed to a `Scanner`, it creates a `Scanner` object that can read keyboard input.

```java
public static void main(String[] args) {  
    Scanner sc = new Scanner(System.in);  
}
```

## next(), nextInt(), nextDouble()

- `next()`: Reads a single word as a `String`, using whitespace as the delimiter.
- `nextInt()`: Reads an integer value.
- `nextDouble()`: Reads a floating-point value.

```java
public static void main(String[] args) {  
    Scanner sc = new Scanner(System.in);  
    System.out.println("Enter your name");  
    String name = sc.next();  
    System.out.println("Enter your age");  
    int age = sc.nextInt();  
    System.out.println("Enter your weight");  
    double weight = sc.nextDouble();  
    System.out.println("Your Name: " + name + "\nYour Age: " +  age + "\nYour Weight: " + weight);  
}
```
```
Outcome

Enter your name
Daniel
Enter your age
23
Enter your weight
72.5
Your Name: Daniel
Your Age: 23
Your Weight: 72.5
```

The `Scanner` class enables users to enter data, which can then be stored and displayed again by the program.

## nextLine()

`next()` and `nextLine()` are both methods of the `Scanner` class used to read `String` input, but they work differently. The `next()` method reads only one word and stops when it encounters whitespace, such as a space, tab, or newline. For example, if the input is `Hello Java`, `next()` reads only `Hello`.

On the other hand, `nextLine()` reads the entire line of input as a `String`. It includes spaces between words and stops only when it reaches the end of the line. Therefore, if the input is `Hello Java`, `nextLine()` reads the full text, `Hello Java`.

One important point is that when methods such as `nextInt()`, `nextDouble()`, or `next()` are used before `nextLine()`, the newline character left in the input buffer may be read by `nextLine()` immediately. As a result, `nextLine()` may return an empty string. To prevent this, an extra `nextLine()` can be used to consume the leftover newline before reading the actual line of input.

## printf()

The `printf()` method is used to print output in a specified format. An important concept here is the format specifier. A format specifier indicates what type of value should be inserted at a specific position in the output. It is written using the `%` symbol.

- `%d`: Specifies the format for printing an integer.
- `%f`: Specifies the format for printing a floating-point
- `%s`: Specifies the format for printing a String.
- `%n`: Represents a line break.

```java
public static void main(String[] args) {  
    System.out.printf("%d%n", 1);  
    System.out.printf("%d %d %d%n", 1, 2, 3);  
}
```
```
Outcome

1
1 2 3
```

## Format Specifier

### Numbers

#### %d

When a number is placed between `%` and `d`, it specifies the width of the output field.

If you want to fill the remaining space with zeros instead of spaces, add 0 before the number.

```java
public static void main(String[] args) {  
    System.out.printf("%d%n", 1234);  
    System.out.printf("%6d%n", 1234);  
    System.out.printf("%06d%n", 1234);  
}
```
```
Outcome

1234
  1234
001234
```

Be careful that the reserved space includes the digits of the integer, not just the spaces before it.

To separate every three digits in a large number, place a comma between `%` and `d`.

``` java
System.out.printf("%,d%n", 1000000);
```
```
Outcome

1,000,000
```

#### %f

To specify the number of digits after the decimal point, place .number between `%` and `f`.

```java
System.out.printf("%.2f", Math.PI);
```
```
Oucome

3.14
```

The other formatting options work the same way as `%d`.

```java
System.out.printf("%8.2f%n", 3.14);
System.out.printf("%08.2f%n", 3.14);
```
```
Outcome

    3.14
00003.14
```

### String

In general, the same formatting rules used for `%d` and `%f` can be applied in the same way.

```java
System.out.printf("%s%n", "Java");
System.out.printf("%6%n", "Java");
System.out.printf("%-6%n", "Java");
```
```
Outcome

Java
  Java
Java
```

Although `String` does not use decimal places, they can still be formatted using `%width precisions`. The number after the dot specifies the maximum number of characters to print from the beginning of the string.

```java
System.out.printf("%6.2s%n", "Java");
```
```
Outcome

    Ja
```


## File

Java allows you to create files, check their information, read and write their contents, and delete them if needed.

```java
public static void main(String[] args) {  
    String fileName = "test.txt";  
    File file = new File(fileName);  
}
```

This code does not create an actual file yet. However, it creates a new `File` object that represents the `test.txt` file inside the program. Later, this object can be used to perform operations such as creating the file, checking file information, or deleting the file.

### createNewFile()

Once a `File` object has been created, the actual file can be created later using the `createNewFile()` method. However, file creation is not guaranteed to always succeed. An exception may occur if there is not enough storage space or if the program does not have permission to create the file, so the code must be wrapped in a `try-catch` block.

```java
public static void main(String[] args) {  
    String fileName = "test.txt";  
    File file = new File(fileName);  
  
    try {  
        file.createNewFile();  
        System.out.println("File created");  
    } catch (IOException e) {  
        throw new RuntimeException(e);  
    }  
}
```

### exists(), getName(), getAbsolutePath(), length()

- `exists()`: Returns `true` if the files exists; otherwise, returns `false`
- `getName()`: Returns the file name
- `getAbsolutePath()`: Returns the absolute path of the file
- `length()`: Returns the file size in bytes

An absolute path is a path that points to the same location regardless of where the program is executed.

```java
public static void main(String[] args) {  
    String fileName = "test.txt";  
    File file = new File(fileName);  
  
    try {  
        file.createNewFile();  
        System.out.println("File created");  
        if (file.exists()) {  
            System.out.println("File Name: " + file.getName());  
            System.out.println("File Location: " + file.getAbsolutePath());  
            System.out.println("File size(Byte): " + file.length());  
        }  
    } catch (IOException e) {  
        throw new RuntimeException(e);  
    }  
}
```
```
Outcome

File created
File Name: test.txt
File Location: /Users/parkcwoo04/Desktop/IntelliJ/JavaWorkspace/test.txt
File size(Byte): 0
```

## Folder

Files are usually managed systemically within a structure called a folder, or directory, rather than existing individually. Just as objects in a room can be organized into boxes by type, folders group related files in one place and help organize the structure of the file system. A file system refers to the overall structure used by a computer to store and organize files and folders.

### mkdir() and mkdirs()

The process of creating a folder is similar to creating a file. As with file creation, you create a `File` object using the `File` class. Then, when you call the `mkdir()` method, a folder is created. `mkdir` stands for "make directory", which means to create a single folder.

```java
public static void main(String[] args) throws IOException {  
    String folderName = "A";  
    File folder = new File(folderName);  
    folder.mkdir();  
    if (folder.exists()) {  
        System.out.println("Folder is created");  
        System.out.println("Location: " + folder.getAbsolutePath());  
    } else {  
        System.out.println("Folder is not created");  
    }  
}
```
```
Outcome

Folder is created
Location: /Users/parkcwoo04/Desktop/IntelliJ/JavaWorkspace/A

```

Programmers can create nested directories at once. For example, if you want to create a folder structure such as `A/B/C`, specify the folder path as `A/B/C` and use the `mkdirs()` method instead of `mkdir()`.

```java
public static void main(String[] args) throws IOException {  
    String folderName = "A/B/C";  
    File folder = new File(folderName);  
    folder.mkdir();  
    if (folder.exists()) {  
        System.out.println("Folder is created");  
        System.out.println("Location: " + folder.getAbsolutePath());  
    } else {  
        System.out.println("Folder is not created");  
    }  
}
```
```
Outcome

Folder is created
Location: /Users/parkcwoo04/Desktop/IntelliJ/JavaWorkspace/A/B/C
```

This is because `mkdir()` method creates a folder only when the parent directory already exists.

## Listing Files and Folders

To list files and folders, you first need to specify the path to the location you want to examine. A folder path can be specified in two ways.

- Relative path: A path based on the current execution location.
- Absolute path: A complete path that starts from the root of the file system.

```java
public static void main(String[] args) throws IOException {  
    String folderPath = ".";  
    File fileOrFolder = new File(folderPath);  
    System.out.println("Current folder location: " + fileOrFolder.getAbsolutePath());  
}
```
```
Outcome

Current folder location: /Users/parkcwoo04/Desktop/IntelliJ/JavaWorkspace/.
```

In code, a period(.) always refers to the current working directory, which is the folder where the code is being executed. After creating a `File` object using this path, you can call the `getAbsolutePath()` method to check the absolute path of that folder.

### listFiles()

The `listFiles()` method of the `File` class allows you to retrieve the files and folders inside a specified directory.


### isFile(), isDirectory()

From the printed list alone, it may be difficult to tell whether each item is a file or a folder. For this purpose, the `File` class provides the following methods

- `isFile()`: Returns `true` if the item is a regular file.
- `isDirectory()`: Returns `true` if the item is a directory.

```java
public static void main(String[] args) throws IOException {  
    String folderPath = "/Users/parkcwoo04/Desktop/IntelliJ/JavaWorkspace";  
    File fileOrFolder = new File(folderPath);  
    System.out.println("=== List of Files ===");  
    for (File file : fileOrFolder.listFiles()) {  
        if (file.isDirectory()) {  
            System.out.println("[Directory] " +  file.getName());  
        } else if (file.isFile()) {  
            System.out.println("[File] " +  file.getName());  
        }  
    }  
}
```

## BufferedReader

`BufferedReader` and `BufferedWriter` are basic tools for reading from and writing to files.

A buffer is a temporary storage area used when reading data from or writing data to a file. Instead of processing data one character at a time, data is collected in a temporary storage space called a buffer and processed together, which greatly improves file input/output performance.

```java
public static void main(String[] args) throws IOException {  
    try (BufferedWriter bw = new BufferedWriter(new FileWriter("test.txt"))) {  
        bw.write("Hello World");  
        bw.newLine();  
        bw.write("Test Sample");  
        bw.newLine();  
    } catch (IOException e) {  
        throw new IOException();  
    }  
}
```

When this code runs, nothing is printed to the console. However, if you open the generated `test.txt` file, you will see that the two sentences written in the code have been saved.

If you run the code above again, the existing file is overwritten instead of appending new content to it. However, if you want to add new content to the end of the file, pass `true` as the second argument when creating the `FileWriter` object.

## BufferedReader

To read an existing file, you first need to connect to the file using a `FileReader` object and the read the actual contents using a `BufferedReader` object.

`FileReader` is responsible for opening the file and connecting it to the program so that its contents can be read. `BufferedReader` allows the program to conveniently read the file contents line by line using the `readLine()` method.

Here, using a `try-with-resources` statement is also safe because the file is automatically closed after the task is completed, so you do not need to call `close()` separately.

```java
public static void main(String[] args) throws IOException {  
    try (BufferedWriter bw = new BufferedWriter(new FileWriter("test.txt", true))) {  
        bw.write("Hello World");  
        bw.newLine();  
        bw.write("Test Sample");  
        bw.newLine();  
    } catch (IOException e) {  
        throw new IOException();  
    }  
  
    try (BufferedReader br = new BufferedReader(new FileReader("test.txt"))) {  
        String line;  
        while ((line = br.readLine()) != null) {  
            System.out.println(line);  
        }  
    }  
}
```
```
Outcome

Hello World
Test Sample
```

## delete()

The most basic way to delete a file is to use the `delete()` method. This method returns `true` if the file is successfully deleted and `false` if the deletion fails.

```java
public static void main(String[] args) throws IOException {  
    File file = new File("test.txt");  
    if (file.delete()) {  
        System.out.println("File deleted successfully");  
    } else {  
        System.out.println("Failed to delete file");  
    }  
}
```
```
Outcome

File deleted successfully
```

However, if you run this code again, a message appears saying that the file deletion failed. This is because the program tried to delete a file that had already been deleted.

Deleting a folder is a little more complicated. If the folder is empty, it can be removed using the delete() method. However, if the folder contains even one file or subfolder, the deletion will fail.

In this case, recursive calls are used. A recursive call is a programming technique in which a method calls itself again inside its won body. The process works as a follows.

1. Go to the folder to be deleted and check the list of files and folders inside it.
2. If there is another folder inside, enter that folder and repeat the same process.
3. Delete the innermost files and folders first.
4. Then move back up to the parent folders and delete them one by one.

```java
public static void main(String[] args) throws IOException {  
    File folder = new File("A");  
    if (deleteFolder(folder)) {  
        System.out.println("File deleted successfully");  
    } else {  
        System.out.println("Failed to delete file");  
    }  
}  
  
public static boolean deleteFolder(File folder) {  
    if (folder.isDirectory()) {  
        for (File file : folder.listFiles()) {  
            deleteFolder(file);  
        }  
    }  
    System.out.println("Deleted the following file: " + folder.getAbsolutePath());  
    return folder.delete();  
}
```
```
Outcome

Deleted the following file: /Users/parkcwoo04/Desktop/IntelliJ/JavaWorkspace/A/B/C
Deleted the following file: /Users/parkcwoo04/Desktop/IntelliJ/JavaWorkspace/A/B
Deleted the following file: /Users/parkcwoo04/Desktop/IntelliJ/JavaWorkspace/A
File deleted successfully
```

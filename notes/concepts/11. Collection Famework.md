
`Collection Framework` enables efficient data management. It dynamically adjusts its capacity according to the number of stored elements and provides built-in methods for operations such as sorting and duplicate elimination.

## List Interface

`List` interface represents a data structure that maintains insertion order and allows duplicate elements. A data structure is a way of storing and organizing dat for efficient use. The `List` interface defines common operations for managing elements, and classes such as `ArrayList` and `LinkedList` provide different implementations optimized for different purposes.

### ArrayList

`ArrayList` is a `List` implementation based on a dynamically resizable array. It provides fast random access because elements can be accessed directly through their indexes.

The internal array automatically grows when additional capacity is needed. However, inserting or removing elements at arbitrary positions requires shifting neighboring elements. Therefore, `ArrayList` is less efficient in workloads involving frequent insertions and deletions.

#### add()

`add()` method is used to add an element to an `ArrayList`. The `ArrayList` stores elements in the order in which they are added.

```java
import java.util.ArrayList;  
  
public class _01_ArrayList {  
    public static void main(String[] args) {  
        ArrayList<String> list = new ArrayList<>();  
        System.out.println("Course registration begins...");  
        list.add("Jake");  
        list.add("Sam");  
        list.add("Sarah");  
        list.add("Kim");  
        list.add("Leah");  
    }  
}
```

- To use `ArrayList`, you must write `import java.util.ArrayList;` at the top of your source code. This import statement tells Java compiler that you intend to use the `ArrayList` class provided by the Java Collections Framework.

- When creating an `ArrayList`, you must declare the element type. In this example, `<String>` specifies that the list stores `String` objects. This is achieved using generics, which enforce type safety.

#### get()

An element in an `ArrayList` can be accessed using the get() method. Each element has an index, and the element at a specific position can be retrieved by passing its index to the method.

```java
list.get(0); // Jake
list.get(1); // Sam
```

#### remove() and size()

To remove a specific element from an `ArrayList`, you can use the `remove()` method. This method allows elements to be deleted easily. In addition, the number of elements stored in the list can be obtained using the `size()` method.

```java
list.size(); // 5
list.remove("Kim"); // remove Kim from the list
list.remove(3); // remove Leah from the list
list.size(); // 3
```

Two elements were removed from the list, reducing its size to three. When an element in the middle of the list is deleted, the subsequent elements are automatically shifted one position to the left. As a result, `Leah`, which was originally at index 4, moves to index 3. Elements can be removed either by specifying their value or by their index.

#### set(), indexOf(), contains()

These methods are used to modify or examine data already stored in a list.

- `set()`: Replaces an existing element with a new one.
- `indexOf()`: Returns the index of a specified element in the list.
- `contains()`: Checks whether a specified element exists in the list and returns a boolean value indicating the result

```java
list.set(0, "Tom"); // Replace Jake to Tom
list.contains("Jake"); // false
list.indexOf("Sam"); // 1
```

#### clear() and isEmpty()

The `clear()` method deletes all elements stored in a list. The `isEmpty()` method checks whether the list contains any elements.

```java
list.clear(); // Clear the list
list.isEmpty() // true
```

#### Collections.sort()

The `Collections` class provides utility methods for working with collections. For example, `Collections.sort()` can sort a list in alphabetical order. To use it, add `import java.util.Collections;` at the beginning of the program.

```java
import java.util.Collections;

Collections.sort(list); // Sort the list in alphabetical order
```

### LinkedList

`ArrayList` is like a long bench where people sit side by side. Adding or removing an element in the middle requires other elements to be shifted, which can reduce performance.

`LinkedList` is like a chain of people holding hands. Elements can be inserted or removed by changing the links between neighboring nodes, making these operations more efficient.

```java
import java.util.LinkedList;  
  
public class _02_LinkedList {  
    public static void main(String[] args) {  
          
        LinkedList<String> list = new LinkedList<>();  
        list.add("Joeun");  
        list.add("Sam");  
        list.add("Kim");  
        list.add("Jan");  
        list.add("Mary");  
    }  
}
```

`LinkedList` has a creation process similar to that of `ArrayList`. Like `ArrayList`, you must import the class, specify the element type using generics, and create an instance using the new keyworkd.

#### getFirst() and getLast()

`LinkedList` allows you to access elements at specific positions using the get() method and an index, just as `ArrayList`. In addition, `LinkedList` provide dedicated methods for retrieving the first and last elements more conveniently.

```java
list.getFirst(); // Joeun
list.getLast(); // Mary
list.get(2); // Kim
```

#### add(), addFirst(), addLast()

By default, the `add()` method adds an  element to the end of the list. To control the position of insertion, you can use: 

- `addFirst()`: Adds an element to the beginning of the list
- `addLast()`: Adds an element to the end of the list
- `add(index, element)`: Inserts and element at the specified index

```java
list.addFirst("Leah"); // Add Leah in the front  
list.addLast("Lee"); // Add Lee in the last  
list.add(3, "Sol"); // Add Sol as the 4th value
```

#### remove(), removeFirst(), removeLast()

The `remove()` method can be used to delete elements stored in a list. Elements may be removed either by specifying their value or their index. Similar to the `add()` method, `LinkedList` provides the `removeFirst()` and `removeLast()` methods, which allow the first and last elements to be removed more intuitively.

```java
list.remove("Lee"); // Remove Lee from the list  
list.remove(0); // Remove Leah from the list  
list.removeFirst(); // Remove Joeun from the list  
list.removeLast(); // Remove Mary from the list.
```

#### set(), indexOf(), contains(), clear(), isEmpty(), Collections.sort()

All of these operations are also available in `LinkedLaist` and are used in a similar manner just like in `ArrayList`.

### ArrayList vs LinkedList

`ArrayList` is optimized for fast data access and is the default choice for most applications. `LinkedList` is optimized for frequent insertions and deletions and is commonly used in queue - like structures. Choosing between them depends on how the data will be accessed and modified.

##### When to Use ArrayList

- Frequent data retrieval is required
- Elements are often accessed by index
- General list operations are needed.
- High performance for searching and reading data is important

##### When to Use LinkedList

- Elements are frequently added or removed at the beginning or end of the list
- Queue or deque structures are required
- The order of elements changes frequently

## Set Interface

Collections in the `List` hierarchy maintain element order and allow duplicate entries. When duplicate elements must be prevented, the `Set` interface is a more appropriate choice. A `Set` ensures that each element is unique and automatically rejects duplicated values.

### HashSet

`HashSet` is a `Set` implementation based on a has table. each element is assigned a hash value through its `hasCode()` method, and this value is used to determine the elements storage location.

As a result, operations such as `add()` , `remove()` and `contains()` are typically very fast. In addition, `HashSet` automatically prevents duplicate elements by comparing hash values and, when necessary, checking equality using the `equials()` method.

While `HashSet` offers excellent performance for uniqueness checks and membership testing, it does not maintain insertion order.

```java
import java.util.HashSet;  
  
public class _03_HashSet {  
    public static void main(String[] args) {  
  
        HashSet<String> set = new HashSet<>();  
        set.add("steak");  
        set.add("coffee");  
        set.add("pizza");  
        set.add("coffee");  
        set.add("pasta");  
        set.add("salad");  
        set.add("coffee");  
        System.out.println("Total Number: " + set.size());  
        System.out.println("=== List of Food ===");  
        for (String food : set) {  
            System.out.println(food);  
        }  
    }  
}
```
```
Outcome

Total Number: 5
=== List of Food ===
pizza
salad
coffee
pasta
steak
```

As shown in the example above, most `HashSet` methods are similar to those provided by the `List` interface.

Although the `add()` method was called seven times, only five elements were actually stored in the collection. This is because the vale `coffee` was added multiple times, and duplicate entries were automatically ignored. As a result, only one instance of `coffee` was retained.

This demonstrates one of the key characteristics of `HasSet`: duplicated elements are automatically eliminated when they are added.

In addition, the output order differs from the order in which the elements were added. This is because `HasSet` does not guarantee insertion order.

### LinkedHashSet

`LinkedHashSet` is a `Set` implementation that combines the fast lookup capabilities of `HashSet` with predictable iteration order. Like `HashSet`, it automatically prevents duplicate elements from being stored. However, unlike `HashSet`, it maintains the order in which elements are added.

The `API` and usage of `LinkedHashSet` are nearly identical to those of `HashSet`.

## Map Interface

Unlike other collection types, a `Map` is a special data structure that stores data as pairs of keys and values. Each key is associated with a corresponding value, allowing data to be retrieved efficiently.

With a `Map`, it is not necessary to search through all stored elements to find specific information. If the appropriate key is known, the associated value can be accessed directly.

An important rule of `Map` is that keys must be unique, whereas values may be duplicated.

### HashMap

`HashMap` is a hash table-based implementation of the `Map` interface. It uses keys to store and retrieve associated values efficiently. 

Operations such as `put()`, `get()`, and `remove()` are typically very fast because they are based on hashing. However, `HashMap` does not guarantee any specific iteration order.

Since keys must be unique, inserting a new value with an exisiting key does not create a duplicate entry. Instead, the existing value associated with that key is overwritten.

#### put()

The `put()` method is used to add data to a `Map` as a key-value pair. When inserting data, both the key and value must match the types specified by the generic parameters declared when the `HashMap` was created.

```java
import java.util.HashMap;  
  
public class _04_HashMap {  
    public static void main(String[] args) {  
        HashMap<String, Integer> map = new HashMap<>();  
          
        map.put("Sarah", 10); // Store Sarah, 10 
        map.put("Kim", 7); // Store Kim, 7
        map.put("Edward", 13); // Store Edward, 13
    }  
}
```

#### size() and get()

These two methods are used to examine data stored in a `Map`. The `size()` method returns the number of key-value pairs stored in the `Map`, while the `get()` method retrieves the value associated with  a specified key.

```java
map.size(); // 3
map.get("Sarah"); // 10
```

A `Map` does not have the concept of indexes. Therefore, `get(index)` cannot be used, and values must always be retrieved using their corresponding keys.

#### containsKey()

The `containsKey()` method is used to determine whether a specific key exists in a `Map`. It returns a boolean value: `true` if the key is present in the map, and `false` otherwise.

```java
map.containsKey("Sarah"); // true
```

#### remove()

The `remove()` method deletes an entry from a `Map` based on its key. When a key is provided, the map removes both key and the value mapped to it. As a result, the corresponding key-value pair no longer exists in the collection.

```java
map.remove("Sarah") // remove Sarah, 10
```

#### clear() and isEmpty()

The `clear()` method removes all entries stored in a Map at once. To verify whether the map is empty after the removal, you can use this `isEmpty()` method.

```java
map.clear(); // remove all datta
map.isEmpty(); // true
```

#### keySet() and values()

The `keySet()` method returns a set containing all keys stored in the map, while the `values()` method returns a collection containing all mapped values.

Although `values()` is useful for processing all values, it does not provide access to associated keys. Therefore, when values are displayed by themselves, it may be unclear which key each value is linked to.

```java
System.out.println(map.keySet());  
System.out.println(map.values());
```
```
Outcome

[Sarah, Edward, Kim]
[10, 13, 7]
```

It also can be printed as

```java
for (String key : map.keySet()) {  
    System.out.println(key);  
}  
System.out.println();  
  
for (int value : map.values()) {  
    System.out.println(value);  
}
```
```
Outcome

Sarah
Edward
Kim

10
13
7
```

As with `HashSet`, the order in which elements are displayed may differ from the order in which they were added. This is because `HashMap` does not guarantee insertion order.

If a map that preserves insertion order is required, `LinkedHashMap` can be used. Unlike `HashMap`, `LinkedHashMap` maintains the order in which entries are added while still providing efficient key-based access. Its usage is virtually identical to that of `HashMap`.

#### Viewing Keys and Values Together

To view both keys and values at the same time, the following basic approach can be used.

```java
for (String key : map.keySet()) {  
    System.out.println(key + ", " + map.get(key));  
}
```
```

```

The `entrySet()` method also provide a view of all entries in a `Map`. Each entry is represented by a `Map`. `Entry` object, which allow access to both the key and its corresponding value through the `getKey()` and `getValue()` methods.

```java
for (HashMap.Entry<String, Integer> entry : map.entrySet()) {  
    System.out.println(entry.getKey() + ", " + entry.getValue());  
}
```

## Iterator

An `Iterator` is sued to iterate through the elements of a collection. It accesses elements one by one using a cursor-like mechanism, regardless of the collection's internal structure.

It is especially useful when removing elements during iteration.

Unlike `List`, `Set`, and `Map`, and `Iterator` is not a collection that stores data. Instead, it is a tool used to traverse the elements stored in a collection one by one.

`Iterator` can be used directly with collections such as `List` and `Set` because they store elements as individual objects. However, a `Map` stores data as key-value pairs rather than single elements, so an `Iterator` cannot be applied directly to a `Map`.

To iterate through the contents of a `Map`, the map must first be converted into a collection view using methods such as `keySet()`, `values()`, or `entrySet()`. An `Iterator` can then be used to traverse the resulting collection.

```java
import java.util.ArrayList;  
import java.util.Iterator;  
  
public class _05_Iterator {  
    public static void main(String[] args) {  
        ArrayList<String> list = new ArrayList<>();  
        list.add("steak");  
        list.add("null");  
        list.add("coffee");  
        list.add("null");  
        list.add("pizza");  
  
        Iterator<String> iterator = list.iterator();  
  
        System.out.println(iterator.next());  
        System.out.println(iterator.next());  
        System.out.println(iterator.next());  
        System.out.println(iterator.next());  
    }  
}
```
```Outcome

steak
null
coffee
null
```

In the code above, an `ArrayList` is used as the underlying collection. To iterate through its elements, an `Iterator` is created by importing the `Iterator` interface and declaring it with the appropriate generic type parameter.

The `next()` method performs two actions:

1. Moves the iterator to the next element.
2. Returns the element at the new cursor position.

Because `next()` is invoked four times in this example, the first four elements of the list are retrieved and printed in their original order.

### Iterating Through All Elements with a Loop

If `next()` continues to be called after the iterator has reached the end of the collection, a runtime error will occur. To prevent this, the `hasNext()` method should be used.

The `hasNext()` method checks whether another element is available and returns a boolean value. When combined with a `while` loop, it allows all elements in a collection to be traversed safely and efficiently.

```java
while(iterator.hasNext()){  
    System.out.println(iterator.next());  
}
```
```
Outcome

steak
null
coffee
null
pizza
```

### remove()

One of the key benefits of using an `Iterator` is its ability to safely modify a collection during traversal. While iterating through the collection, elements that match certain criteria can be removed using the iterator’s `remove()` method.

This helps avoid runtime exceptions that may occur when modifying a collection directly during iteration.

```java
while(iterator.hasNext()){  
    if(iterator.next().equals("null")){  
        iterator.remove();  
    }  
}
```

All the `null` from the list has been removed.

### Using an Iterator with a Map

In a `Set`, an `Iterator` can be used in the same way as it is used with a `List`. However, using an `Iterator` with a `Map` is slightly more complex.

Iterating through only the keys or only the values is relatively straightforward. However, when working with key-value pairs together, the process of creating and using an `Iterator` is somewhat different.

```java
Iterator<HashMap.Entry<String, Integer>> iterator = list.entrySet().iterator();  
  
while(iterator.hasNext()){  
   System.out.println(iterator.next());  
}
```
```
Outcome

pizza=12
steak=10
coffee=7
```

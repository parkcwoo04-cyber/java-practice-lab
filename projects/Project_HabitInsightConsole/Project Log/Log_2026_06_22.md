## 4. Current Functional Flow

The current console program now supports this full first-version workflow:

```text
Add Habit
→ Search Habit
→ Remove Habit
→ Manage Habit
→ Mark Habit as Completed
→ Print All Habits
→ View Summary Report
```

The newly added progress tracking flow works like this:

```text
Progress Tracking
→ Enter habit title keyword
→ Display matching habits
→ Select habit number
→ Run markComplete()
→ Update completed count, streak, and completion rate
```

This means the project has moved from “habit creation only” to an actual habit tracking console application.

## 5. Issues / Bugs Found

### 1. Console class structure needs cleanup

The `HabitInsightConsole` class is becoming too long because most menu handling logic is still inside the main execution flow.

Next improvement should separate logic into helper methods such as:

```text
handleHabitManagementMenu()
handleSearchMenu()
handleProgressTrackingMenu()
handleReportsMenu()
handleAddHabit()
handleRemoveHabit()
handleManageHabit()
printHabitResults()
```

### 2. Input validation still needs improvement

Most numeric input currently depends on `nextInt()`. If the user enters text instead of a number, the program may still crash.

This should be improved later with safer input handling.

### 3. Repeated search result printing should be refactored

Search, remove, manage, and progress tracking all need to display habit search results. If similar printing logic is repeated in multiple places, it should be extracted into one reusable method.

### 4. Progress tracking edge cases need testing

The `markComplete()` feature is now connected, but it should still be tested carefully.

Important cases:

```text
First completion
Duplicate completion on the same day
Completion after yesterday
Completion after missing several days
Completion attempt on paused or archived habit
```

### 5. Summary report should be tested with multiple habit types

The report should be tested with study, exercise, and general habits across different categories and statuses to confirm that statistics are calculated correctly.
